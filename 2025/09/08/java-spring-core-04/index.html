Spring 核心 04 - Bean 的完整生命週期

發表於2025-09-08|更新於2025-09-29|JavaSpringSpring 核心實戰
|字數總計:1.7k|閱讀時長:5分鐘|閱讀量:

前言：Bean 的一生

如果說 Bean 的作用域 (Scope) 定義了它在公司裡的「聘僱合約」（是終身職 singleton，還是臨時工 request），那麼 Bean 的**生命週期 (Lifecycle)**，則描述了它從「到職」到「離職」所經歷的完整流程，包括了入職培訓（初始化）和離職交接（銷毀）。

我們常常會有這樣的需求：當一個 Bean 被建立且所有必要的「工具」（依賴）都分配好之後，需要立刻執行一些初始化任務，例如：

  • 預先從資料庫載入一些基礎資料到快取中。
  • 建立一個網路連線。
  • 讀取一個外部設定檔。

同樣地，當應用程式要關閉時，我們也希望這個 Bean 能優雅地釋放它所持有的資源，例如關閉檔案、中斷連線等。

直接在建構子 (Constructor) 中執行初始化邏輯，往往不是一個好主意，因為在建構子執行時，Spring 可能還沒有將所有依賴注入進來。為此，Spring 提供了可靠的生命週期回呼 (Lifecycle Callbacks) 機制，讓我們可以在 Bean 生命的關鍵時刻，掛載上我們自訂的邏輯。本篇將重點介紹最常用、最推薦的標準作法：@PostConstruct 與 @PreDestroy。

Bean 生命週期的簡化視圖

對於一個 singleton 的 Bean，它在 Spring IoC 容器中的一生，大致會經歷以下幾個關鍵階段：

  1. **實例化 (Instantiation)**：IoC 容器找到 Bean 的定義，透過 Java 反射機制呼叫其建構子，建立出一個物件實例。此刻，它還只是一個「空的軀殼」。
  2. **屬性填充 (Populate Properties)**：容器根據 @Autowired 或建構子等定義，將這個 Bean 所需的所有依賴（其他的 Bean）注入進來。此刻，這位「員工」才算是領到了所有「工具」。
  3. 初始化 (Initialization)：在 Bean 完全準備就緒之後，容器會呼叫其設定好的初始化方法。@PostConstruct 註解的方法就在這個階段被執行。此刻，Bean 才真正「準備上工」。
  4. **使用中 (In Use)**：應用程式正常運行，這個 Bean 被其他物件呼叫，執行其業務職責。
  5. 銷毀 (Destruction)：當 IoC 容器關閉時（例如應用程式正常關閉），容器會呼叫其設定好的銷毀方法。@PreDestroy 註解的方法就在這個階段被執行，用於資源清理。

初始化回呼：@PostConstruct

@PostConstruct 是一個 JSR-250 標準註解，它標示的方法將會在所有依賴注入完成之後被自動呼叫。這是執行初始化邏輯最安全、最可靠的時機。

銷毀回呼：@PreDestroy

@PreDestroy 同樣是一個 JSR-250 標準註解，它標示的方法將會在容器銷毀 Bean 之前被呼叫。這給了我們一個機會來執行清理工作。

重要提示：銷毀回呼僅對 singleton 作用域的 Bean 有效。對於 prototype Bean，容器在將其交給客戶端後便不再管理，因此不會呼叫其銷毀方法。

其他生命週期回呼方式

除了註解，還有其他兩種歷史更悠久的方式，您也應該有所了解：

  1. 實現介面：讓您的 Bean 實現 Spring 的 InitializingBean 和 DisposableBean 介面，並實作其 afterPropertiesSet() 和 destroy() 方法。這種方式的缺點是讓您的業務程式碼與 Spring 框架產生了耦合。
  2. @Bean 的屬性：在 @Configuration 類別中定義 @Bean 時，可以指定 initMethod 和 destroyMethod 屬性。 這種方式在為第三方函式庫的類別（您無法修改其原始碼來加上註解）定義 Bean 時非常有用。

總結

在本篇文章中，我們一起探索了 Spring Bean 從誕生到消亡的完整旅程。

  • 我們了解了 Bean 生命週期中的實例化、屬性填充、初始化、使用、銷毀等關鍵階段。
  • 我們重點掌握了 JSR-250 標準提供的 @PostConstruct 和 @PreDestroy 註解，這是進行初始化與銷毀操作的現代化最佳實踐。
  • 我們也了解了其他兩種回呼方式，以應對不同的場景。

掌握 Bean 的生命週期，能讓您撰寫出更穩健、資源管理更完善的元件。
